Cliente GUI JavaFX para Microservicio JWT

1. Objetivo del Proyecto

Desarrollar un cliente JavaFX moderno para consumir el microservicio JWT. La aplicación proporciona una interfaz gráfica nativa con autenticación completa, gestión de tokens JWT y operaciones protegidas, utilizando las mejores prácticas de desarrollo Java con Maven.

Características principales:

  • Interfaz JavaFX nativa moderna y responsiva
  • Cliente HTTP OkHttp para comunicación REST
  • Procesamiento JSON con Jackson
  • Gestión completa de tokens JWT (solo en Redis)
  • Configuración persistente en archivo JSON local
  • Validación de formularios y feedback visual
  • Manejo de errores robusto y logging
  • Arquitectura MVC con FXML
2. Tecnologías Utilizadas
Java JavaFX Maven JWT OkHttp Jackson

JavaFX Framework

  • Java 17+ - JDK moderno con soporte JavaFX
  • JavaFX - Framework para aplicaciones desktop nativas
  • FXML - Lenguaje declarativo para interfaces

Herramientas y Librerías

  • Maven - Gestión de dependencias y build
  • OkHttp - Cliente HTTP moderno
  • Jackson - Procesamiento JSON
3. Arquitectura del Sistema

La arquitectura sigue el patrón MVC (Model-View-Controller) con separación clara entre la interfaz gráfica, la lógica de negocio y la comunicación con el microservicio.

JavaFX GUI

Interfaz gráfica nativa que maneja todos los eventos del usuario

OkHttp Client

Cliente HTTP para comunicación REST con el microservicio

JSON Config

Configuración persistente de conexión y endpoints

4. Prerrequisitos
  • JDK 17 o superior (con soporte JavaFX)
  • Maven 3.6 o superior
  • Microservicio JWT corriendo (puerto 5000 por defecto)
  • Puertos disponibles: 5000 (Flask), 3306 (MariaDB), 6379 (Redis)
5. Instalación
  1. Clona el repositorio 
    git clone https://github.com/bryramirezp/Cliente-Java-GUI-Microservicio-JWT.git
cd Cliente-Java-GUI-Microservicio-JWT
  2. Compila el proyecto 
    mvn clean compile
  3. Ejecuta la aplicación 
    mvn javafx:run
6. Configuración

Archivo de Configuración (jwt_gui_config.json)

La aplicación crea automáticamente un archivo jwt_gui_config.json para almacenar la configuración de conexión:

{
  "ip": "localhost",
  "port": "5000",
  "endpoints": {
    "register": "/register",
    "login": "/login",
    "protected": "/protected",
    "refresh": "/refresh",
    "logout": "/logout",
    "users": "/users"
  }
}

Nota Importante sobre Tokens

🔒 Los tokens JWT se gestionan únicamente en Redis

La aplicación JavaFX no almacena tokens localmente. Todos los tokens se mantienen en el servidor Redis del microservicio, proporcionando mayor seguridad y consistencia.

Configuración Inicial

  • IP: Dirección del microservicio (por defecto: localhost)
  • Puerto: Puerto del microservicio (por defecto: 5000)
  • Endpoints: URLs de los servicios REST
  • Tokens: Gestión exclusiva en Redis (no local)
7. Uso

Interfaz de la GUI JavaFX

La aplicación se divide en secciones organizadas para facilitar la interacción:

  • Configuración del Microservicio: Campos para IP y puerto
  • Registro de Usuario: Campos para crear nueva cuenta
  • Inicio de Sesión: Campos para autenticación
  • Acciones Autenticadas: Botones para operaciones con tokens
  • Gestión de Usuarios: Ver y gestionar usuarios
  • Estado de Conexión: Indicadores visuales de conectividad

Funcionalidades Principales

Registro de Usuario

  • Campos: Usuario, Email, Contraseña
  • Validación de campos requeridos
  • Feedback visual de éxito/error

Login

  • Campos: Usuario, Contraseña
  • Tokens gestionados en Redis
  • Estado de autenticación visual

Acceso Protegido

  • Requiere token de acceso válido
  • Validación automática contra Redis

Refresh Token

  • Utiliza refresh token para renovar acceso
  • Gestión automática de expiración

Logout

  • Revoca tokens en el servidor
  • Limpieza completa de sesión
