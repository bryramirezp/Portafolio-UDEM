Ejercicio Guiado 4

Autenticación JWT en Microservicios

Descripción de la Actividad

Microservicio Flask con JWT

La actividad consiste en desarrollar un microservicio en Flask que use JWT (JSON Web Token) para autenticar usuarios contra una base de datos. El sistema debe permitir:

  • Registrar usuarios y permitir inicio de sesión.
  • Generar tokens JWT al autenticarse correctamente.
  • Proteger rutas de la API para que solo se accedan con un token válido.
  • Crear una aplicación web (HTML, CSS, JS) separada que consuma el microservicio, almacenando el token en el navegador y enviándolo en las cabeceras de cada petición.
  • Incluir logs en el servidor y en el cliente para observar el flujo de autenticación.

Flujo General de JWT

  1. El usuario envía sus credenciales al servidor.
  2. El servidor, si son válidas, genera un JWT con información del usuario y lo firma.
  3. El JWT se envía al cliente.
  4. El cliente lo almacena (ej. localStorage).
  5. En cada petición a un recurso protegido, el cliente envía el JWT en la cabecera.
  6. El servidor verifica la firma del JWT.
  7. Si es válido, procesa la solicitud sin pedir credenciales de nuevo.
