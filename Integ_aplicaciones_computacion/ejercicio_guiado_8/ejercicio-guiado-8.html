Ejercicio Guiado 8

Microservicio JWT con Swagger

1. Objetivo del Proyecto

Desarrollar un microservicio completo en Python utilizando Flask como framework web, implementando autenticación JWT (JSON Web Tokens) con Redis para gestión de sesiones, MariaDB como base de datos relacional, y documentación automática con Swagger/OpenAPI 2.0 a través de Flasgger. El proyecto demuestra las mejores prácticas en desarrollo de APIs REST seguras, documentadas y escalables.

Características principales:

  • Autenticación completa con JWT (registro, login, refresh, logout)
  • Documentación interactiva automática con Swagger UI
  • Endpoints REST protegidos con validación de tokens
  • Gestión de sesiones con Redis como almacén temporal
  • Base de datos MariaDB para persistencia de usuarios
  • Validación automática de esquemas con Flasgger
  • Contenedorización con Docker y docker-compose
  • Monitoreo de salud del servicio

4. Prerrequisitos
  • Python 3.9 o superior
  • Docker y docker-compose
  • Puertos disponibles: 5000 (Flask), 3306 (MariaDB), 6379 (Redis)
  • Git para clonar el repositorio
5. Instalación
  1. Clona el repositorio 
    git clone https://github.com/bryramirezp/microservicio-auth-swagger-jwt.git
cd microservicio-auth-swagger-jwt
  2. Con Docker (recomendado) 
    docker-compose up --build
  3. Sin Docker (desarrollo) 
    pip install -r requirements.txt
python app.py
6. Configuración

Variables de Entorno

El proyecto utiliza variables de entorno para configuración flexible:

# Flask
FLASK_ENV=development
FLASK_DEBUG=True

# Base de datos
MYSQL_HOST=mariadb
MYSQL_USER=root
MYSQL_PASSWORD=password
MYSQL_DB=auth_db

# Redis
REDIS_HOST=redis
REDIS_PORT=6379

# JWT
JWT_SECRET_KEY=your-secret-key-here
JWT_ACCESS_TOKEN_EXPIRE_MINUTES=15
JWT_REFRESH_TOKEN_EXPIRE_DAYS=7

Configuración Docker

El archivo docker-compose.yml define tres servicios:

  • web: Aplicación Flask con Flasgger
  • mariadb: Base de datos MariaDB
  • redis: Almacén de sesiones Redis
7. Uso

Acceso a Swagger UI

Una vez iniciado el servicio, accede a la documentación interactiva:

http://localhost:5000/apidocs/

¿Qué es Swagger?

Swagger es una herramienta poderosa para diseñar, construir, documentar y consumir APIs REST. Utilizamos Flasgger para integrar Swagger/OpenAPI 2.0 directamente en nuestro microservicio.

Interfaz Swagger UI

Interfaz interactiva de Swagger UI para probar y documentar la API

Beneficios de Swagger

  • Documentación Interactiva: Interfaz web para explorar y probar endpoints
  • Esquemas de Seguridad: Define claramente el uso de tokens JWT
  • Validación Automática: Decoradores validan respuestas con esquemas
  • Descubrimiento de APIs: Desarrolladores entienden APIs rápidamente

Flujo de Uso Típico

1. Registro

POST /register - Crear nueva cuenta de usuario

{
  "username": "testuser",
  "email": "test@example.com",
  "password": "securepass123"
}

2. Login

POST /login - Obtener tokens JWT

{
  "username": "testuser",
  "password": "securepass123"
}

3. Autorización

Usar token en header Authorization:

Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...

4. Acceso Protegido

GET /protected - Requiere token válido

✅ Respuesta exitosa con datos protegidos

  • Hashing de Contraseñas: Uso de werkzeug.security para hashing seguro

  • Revocación de Tokens: Sistema de blacklist con Redis

Documentación

  • Swagger/OpenAPI: Documentación automática e interactiva
  • Esquemas de Respuesta: Definición clara de todas las respuestas HTTP

Arquitectura
